# aispeak

BiliGlass Subtitle Extraction 是一个面向 B 站视频转写与字幕整理的工具集，既能处理在线 BV/AV 视频，也支持本地音视频文件转写。它的目标很直接：把“找字幕、转音频、拆时间轴、整理文本”这些重复劳动压缩成一次点击，让内容整理回到内容本身。

这个仓库同时保留了桌面 GUI 和 Web 入口，适合不同使用习惯：

- 想点一下就跑，打开 `start.bat`
- 想在浏览器里管理任务，打开 `start_web.bat`
- 想处理本地 `mp3/mp4/m4a/wav/flac/aac/mov/mkv/avi/webm`，也可以直接上传文件或文件夹

所有转写结果默认会统一为简体中文，避免输出风格混乱。

## 核心功能

- B 站视频转写：输入 BV/AV视频链接即可处理
- 本地媒体转写：支持单个文件或当前目录文件夹，不递归子目录
- AI 字幕提取：优先获取可用的 B 站 AI 字幕
- Whisper 转写：在没有 AI 字幕时，使用本地 Whisper 进行语音识别
- 简体中文输出：字幕与转写文本都会统一转换为简体中文
- 双入口使用：同时提供桌面 GUI 和 Web 页面
- 任务队列管理：可查看任务状态、进度、错误信息和输出结果

## 项目亮点

如果把这个项目比作一个字幕工厂，它做的不是“展示一个页面”，而是把一整条工作流串起来：

1. 发现视频或本地文件
2. 获取音频或字幕
3. 进行识别与整理
4. 输出成可直接阅读和复用的文本

这意味着你不需要分别打开下载器、转写器、字幕整理器和文件管理器。一个入口就够了。

## 环境要求

### 运行环境

- Windows 10 / 11
- Python 3.10 或更高版本，建议 3.11+
- 建议使用独立虚拟环境或 Conda 环境

### 依赖包

核心依赖包括：

- `customtkinter`
- `bilibili-api`
- `aiohttp`
- `faster-whisper`
- `opencc-python-reimplemented`
- `torch`，如需 GPU 加速则安装对应 CUDA 版本

### 系统依赖

- `ffmpeg`：用于音频处理和转写流程
- 可访问哔哩哔哩网络接口
- 如需拉取视频字幕或音频，需配置有效的 B 站 Cookie

### 存储建议

- 建议保留足够磁盘空间用于音频缓存和转写结果
- 默认输出目录为 `videos/`

## 快速开始

### 1. 准备环境

1. 安装 Python 3.10+。
2. 安装项目依赖。
3. 确保 `ffmpeg` 可用。
4. 如需处理 B 站内容，准备好 Cookie。

### 2. 启动桌面 GUI

双击根目录下的 `start.bat`。

### 3. 启动 Web 页面

双击根目录下的 `start_web.bat`。

浏览器会自动打开控制台页面，你也可以在页面里切换到本地文件转写相关入口。

### 4. 运行测试脚本

测试脚本位于 `tests/` 目录下，可直接执行：

```bash
python tests/test_bilibili.py
```

## 使用说明

### 在线视频处理

1. 在 GUI 或 Web 控制台中输入 BV 号或视频链接。
2. 系统会自动解析视频信息并加入队列。
3. 如果有可用 AI 字幕，会优先提取字幕。
4. 如果没有可用字幕，则使用 Whisper 进行转写。

### 本地文件处理

1. 选择一个本地音视频文件，或选择当前目录下的一组文件。
2. 系统会按文件分别生成转写结果。
3. 输出文件会写入 `videos/` 及其子目录。

支持的常见格式：

- 音频：`mp3`、`m4a`、`wav`、`flac`、`aac`
- 视频：`mp4`、`mov`、`mkv`、`avi`、`webm`

### Cookie 管理

- 如果提示 Cookie 未配置，需要先在 GUI 或 Web 中补充 Cookie。
- Cookie 只用于访问 B 站相关接口，不应上传到仓库。

## 输出位置

- 转写结果默认保存在 `videos/`
- 临时音频和中间文件会放在 `videos/temp_audio/` 等子目录
- 字幕整理结果通常会放在 `videos/subtitles/`

## 常见问题

### 为什么输出是简体中文？

因为转写与字幕整理流程里统一做了繁简转换，避免不同来源的文本风格不一致。

### 为什么有 GUI 还要 Web？

GUI 适合快速本地操作，Web 适合在浏览器里查看任务、管理配置和处理更复杂的工作流。

### 可以直接上传 cookies 吗？

不可以。Cookie 属于敏感信息，不应提交到版本库，也不应上传到公开仓库。
